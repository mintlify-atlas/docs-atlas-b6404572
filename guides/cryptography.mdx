## Supported algorithms

libspdm supports the following algorithm families, negotiated during `NEGOTIATE_ALGORITHMS`.

### Hash algorithms

| Algorithm | Notes |
|-----------|-------|
| SHA-256, SHA-384, SHA-512 | SHA-2 family |
| SHA3-256, SHA3-384, SHA3-512 | SHA-3 family |
| SM3-256 | Chinese national standard |

### Asymmetric signature algorithms

| Algorithm | Notes |
|-----------|-------|
| RSA-SSA (PKCS#1 v1.5) | 2048, 3072, 4096-bit keys |
| RSA-PSS | 2048, 3072, 4096-bit keys |
| ECDSA | P-256, P-384, P-521 |
| EdDSA | Ed25519, Ed448 |
| SM2-Sign | Chinese national standard |
| ML-DSA | Post-quantum; RFC 9881 (ML-DSA-44, ML-DSA-65, ML-DSA-87) |
| SLH-DSA | Post-quantum; RFC 9909 |

### Key exchange algorithms

| Algorithm | Notes |
|-----------|-------|
| FFDHE | 2048, 3072, 4096-bit groups |
| ECDHE | P-256, P-384, P-521 |
| SM2-KeyExchange | Chinese national standard |
| ML-KEM | Post-quantum (ML-KEM-512, ML-KEM-768, ML-KEM-1024) |

### AEAD cipher suites

| Algorithm | Notes |
|-----------|-------|
| AES-128-GCM, AES-256-GCM | |
| ChaCha20-Poly1305 | |
| SM4-GCM | Chinese national standard |

## Post-quantum cryptography

libspdm includes support for NIST-standardized post-quantum algorithms:

- **ML-DSA** (Module-Lattice Digital Signature Algorithm) — standardized in [RFC 9881](https://www.rfc-editor.org/rfc/rfc9881). Supports parameter sets ML-DSA-44, ML-DSA-65, and ML-DSA-87.
- **SLH-DSA** (Stateless Hash-Based Digital Signature Algorithm) — standardized in [RFC 9909](https://www.rfc-editor.org/rfc/rfc9909).
- **ML-KEM** (Module-Lattice Key Encapsulation Mechanism) — supports ML-KEM-512, ML-KEM-768, and ML-KEM-1024.

PQC algorithm support requires the OpenSSL backend (see backend comparison below). Enable
the specific parameter sets via the corresponding `spdm_lib_config.h` macros before
building.

## Crypto backends

libspdm provides two cryptography backends. Choose one at build time.

### OpenSSL wrapper (`cryptlib_openssl`)

The OpenSSL wrapper supports all algorithm families listed above with the following
exceptions:

- **SM2-KeyExchange** is not supported.
- **SM4-GCM** is not supported.

Use the OpenSSL wrapper when you require EdDSA, ML-DSA, SLH-DSA, or ML-KEM support.

### Mbed TLS wrapper (`cryptlib_mbedtls`)

The Mbed TLS wrapper has a more limited algorithm set. The following are **not** supported:

- SM2, SM3, SM4 (all SM-family algorithms)
- EdDSA (Ed25519, Ed448)
- ML-DSA
- SLH-DSA
- ML-KEM

Use the Mbed TLS wrapper for constrained environments where OpenSSL is not available, and
only when the unsupported algorithms are not required.

## Selecting algorithms via libspdm_set_data

Algorithm selection is communicated to the SPDM peer during `NEGOTIATE_ALGORITHMS`. Set
the desired values on the local context using `libspdm_set_data` before calling
`libspdm_init_connection`.

```c
parameter.location = LIBSPDM_DATA_LOCATION_LOCAL;

/* Base asymmetric signature algorithm (e.g. SPDM_ALGORITHMS_BASE_ASYM_ALGO_TPM_ALG_ECDSA_ECC_NIST_P256) */
libspdm_set_data(spdm_context, LIBSPDM_DATA_BASE_ASYM_ALGO,
                 &parameter, &base_asym_algo, sizeof(base_asym_algo));

/* Base hash algorithm (e.g. SPDM_ALGORITHMS_BASE_HASH_ALGO_TPM_ALG_SHA_256) */
libspdm_set_data(spdm_context, LIBSPDM_DATA_BASE_HASH_ALGO,
                 &parameter, &base_hash_algo, sizeof(base_hash_algo));

/* DHE named group for key exchange (e.g. SPDM_ALGORITHMS_DHE_NAMED_GROUP_SECP_256_R1) */
libspdm_set_data(spdm_context, LIBSPDM_DATA_DHE_NAME_GROUP,
                 &parameter, &dhe_named_group, sizeof(dhe_named_group));

/* AEAD cipher suite (e.g. SPDM_ALGORITHMS_AEAD_CIPHER_SUITE_AES_256_GCM) */
libspdm_set_data(spdm_context, LIBSPDM_DATA_AEAD_CIPHER_SUITE,
                 &parameter, &aead_cipher_suite, sizeof(aead_cipher_suite));
```

For mutual authentication, also set the Requester's asymmetric algorithm:

```c
/* Requester base asymmetric algorithm (e.g. SPDM_ALGORITHMS_BASE_ASYM_ALGO_TPM_ALG_ECDSA_ECC_NIST_P256) */
libspdm_set_data(spdm_context, LIBSPDM_DATA_REQ_BASE_ASYM_ALG,
                 &parameter, &req_base_asym_alg, sizeof(req_base_asym_alg));
```

## Key configuration macros

Enable specific algorithm parameter sets in `include/library/spdm_lib_config.h`. Only
algorithms enabled here are compiled in and available for negotiation.

```c
/* RSA signature variants */
#define LIBSPDM_RSA_SSA_2048_SUPPORT 1
#define LIBSPDM_RSA_SSA_3072_SUPPORT 1
#define LIBSPDM_RSA_SSA_4096_SUPPORT 1
#define LIBSPDM_RSA_PSS_2048_SUPPORT 1
#define LIBSPDM_RSA_PSS_3072_SUPPORT 1
#define LIBSPDM_RSA_PSS_4096_SUPPORT 1

/* ECDHE key exchange groups */
#define LIBSPDM_ECDHE_P256_SUPPORT 1
#define LIBSPDM_ECDHE_P384_SUPPORT 1
#define LIBSPDM_ECDHE_P521_SUPPORT 1

/* ECDSA signature variants */
#define LIBSPDM_ECDSA_P256_SUPPORT 1
#define LIBSPDM_ECDSA_P384_SUPPORT 1
#define LIBSPDM_ECDSA_P521_SUPPORT 1

/* Post-quantum */
#define LIBSPDM_ML_DSA_44_SUPPORT 1
#define LIBSPDM_ML_DSA_65_SUPPORT 1
#define LIBSPDM_ML_DSA_87_SUPPORT 1
#define LIBSPDM_ML_KEM_512_SUPPORT  1
#define LIBSPDM_ML_KEM_768_SUPPORT  1
#define LIBSPDM_ML_KEM_1024_SUPPORT 1
```

Disabling unused algorithms reduces code size and the attack surface of the build.
