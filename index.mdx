libspdm provides everything you need to implement SPDM-based device authentication, attestation, and secure communication in firmware, drivers, and embedded systems. It supports SPDM specifications 1.0 through 1.4 and runs without dynamic memory allocation, making it suitable for resource-constrained environments.

## What is SPDM?

The Security Protocol and Data Model (SPDM) is a DMTF standard (DSP0274) that enables authenticated communication between hardware components. It provides:

- **Device authentication** — verify the identity of a device using certificates
- **Attestation** — retrieve and verify signed measurements of device firmware and configuration
- **Secure sessions** — establish encrypted channels between Requester and Responder endpoints

libspdm implements both sides of the protocol — the **Requester** (initiates requests) and the **Responder** (handles requests and provides measurements/certificates).

## Get started

<Steps>
  <Step title="Build the library">
    Clone the repository, initialize submodules, and build with CMake. See the [build guide](/build) for all platform and toolchain options.
  </Step>
  <Step title="Initialize an SPDM context">
    Allocate and initialize an `spdm_context`, register device I/O callbacks, and configure capabilities and algorithms.
  </Step>
  <Step title="Connect and authenticate">
    Call `libspdm_init_connection` to negotiate version and capabilities, then use `libspdm_get_digest`, `libspdm_get_certificate`, and `libspdm_challenge` to authenticate the peer.
  </Step>
  <Step title="Retrieve measurements or start a session">
    Use `libspdm_get_measurement` for attestation, or `libspdm_start_session` to establish an encrypted SPDM session.
  </Step>
</Steps>
