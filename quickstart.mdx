## Prerequisites

Before building libspdm, ensure the following tools are installed:

- **Git** with submodule support
- **CMake** (version 3.17.2 known to work; newer versions may fail)
- **Perl** 5.10.0 or higher (required for building OpenSSL from source)
- A supported C compiler — GCC 5+ or Clang 10+ on Linux; Visual Studio 2019/2022 or LLVM 13 on Windows
- **cmocka** 1.1.7 (for unit tests; included as a submodule)
- A crypto library — Mbed TLS 3.6.5 or OpenSSL 3.5.5 (included as submodules)

<Steps>

### Clone and initialize submodules

libspdm uses Git submodules for Mbed TLS, OpenSSL, and cmocka. Initialize them after cloning.

```bash
git clone https://github.com/DMTF/libspdm.git
cd libspdm
git submodule update --init
```

### Build libspdm

Choose the build commands for your platform.

<CodeGroup>

```bash Linux (x64, GCC, OpenSSL, Debug)
mkdir build && cd build
cmake -DARCH=x64 -DTOOLCHAIN=GCC -DTARGET=Debug -DCRYPTO=openssl ..
make copy_sample_key
make
```

```bash Windows (x64, VS2019, OpenSSL, Debug)
mkdir build
cd build
cmake -G"NMake Makefiles" -DARCH=x64 -DTOOLCHAIN=VS2019 -DTARGET=Debug -DCRYPTO=openssl ..
nmake copy_sample_key
nmake
```

</CodeGroup>

The `copy_sample_key` target copies pre-generated test certificates into the build tree so that unit tests and the emulator can run without requiring a PKI setup.

### Run the unit tests

After a successful build, the test binaries are placed in `build/bin/`.

```bash
cd build/bin
./test_spdm_requester
./test_spdm_responder
```

Expected output:

```
[==========] Running 2 test(s).
[ RUN      ] test_spdm_responder_version_case1
[       OK ] test_spdm_responder_version_case1
[ RUN      ] test_spdm_responder_version_case2
[       OK ] test_spdm_responder_version_case2
[==========] 2 test(s) run.
[  PASSED  ] 2 test(s).
```

<Note>
You must run the unit tests from the `build/bin` directory. Running them from the parent `build/` directory (e.g., `bin/test_spdm_responder`) may cause tests to fail because they look for test data relative to the current working directory.
</Note>

### Write a minimal Requester

The following example shows the minimum code required to initialize an SPDM context and establish a connection with a Responder.

```c
#include "library/spdm_requester_lib.h"
#include "library/spdm_transport_mctp_lib.h"

/* Step 1: Allocate and initialize the SPDM context. */
void *spdm_context = malloc(libspdm_get_context_size());
libspdm_init_context(spdm_context);

/* Set up the scratch buffer (used for decrypted secured messages). */
size_t scratch_buffer_size = libspdm_get_sizeof_required_scratch_buffer(spdm_context);
void *scratch_buffer = malloc(scratch_buffer_size);
libspdm_set_scratch_buffer(spdm_context, scratch_buffer, scratch_buffer_size);

/* Step 2: Register device I/O callbacks (provided by your transport driver). */
libspdm_register_device_io_func(
    spdm_context,
    spdm_device_send_message,    /* your send implementation */
    spdm_device_receive_message  /* your receive implementation */
);

/* Step 3: Register the transport layer (MCTP shown here). */
libspdm_register_transport_layer_func(
    spdm_context,
    LIBSPDM_MAX_SPDM_MSG_SIZE,
    LIBSPDM_MCTP_TRANSPORT_HEADER_SIZE,
    LIBSPDM_MCTP_TRANSPORT_TAIL_SIZE,
    libspdm_transport_mctp_encode_message,
    libspdm_transport_mctp_decode_message
);

/* Step 4: Register device buffer callbacks. */
libspdm_register_device_buffer_func(
    spdm_context,
    LIBSPDM_SENDER_BUFFER_SIZE,
    LIBSPDM_RECEIVER_BUFFER_SIZE,
    spdm_device_acquire_sender_buffer,
    spdm_device_release_sender_buffer,
    spdm_device_acquire_receiver_buffer,
    spdm_device_release_receiver_buffer
);

/* Step 5: Configure capabilities and algorithm preferences. */
libspdm_data_parameter_t parameter;
parameter.location = LIBSPDM_DATA_LOCATION_LOCAL;

libspdm_set_data(spdm_context, LIBSPDM_DATA_CAPABILITY_CT_EXPONENT,
                 &parameter, &ct_exponent, sizeof(ct_exponent));
libspdm_set_data(spdm_context, LIBSPDM_DATA_CAPABILITY_FLAGS,
                 &parameter, &cap_flags, sizeof(cap_flags));
libspdm_set_data(spdm_context, LIBSPDM_DATA_BASE_ASYM_ALGO,
                 &parameter, &base_asym_algo, sizeof(base_asym_algo));
libspdm_set_data(spdm_context, LIBSPDM_DATA_BASE_HASH_ALGO,
                 &parameter, &base_hash_algo, sizeof(base_hash_algo));

/* Step 6: Initiate the SPDM connection.
 * Sends GET_VERSION, GET_CAPABILITIES, and NEGOTIATE_ALGORITHMS. */
libspdm_return_t status = libspdm_init_connection(spdm_context, false);
if (LIBSPDM_STATUS_IS_ERROR(status)) {
    /* handle connection error */
}

/* Clean up when done. */
libspdm_deinit_context(spdm_context);
free(scratch_buffer);
free(spdm_context);
```

</Steps>

## Next steps

After establishing a connection you can authenticate the Responder, retrieve measurements, and establish encrypted sessions:

- Call `libspdm_get_digest`, `libspdm_get_certificate`, and `libspdm_challenge` to authenticate the Responder.
- Call `libspdm_get_measurement` to retrieve firmware measurements.
- Call `libspdm_start_session` to establish an encrypted SPDM session using `KEY_EXCHANGE`/`FINISH` or `PSK_EXCHANGE`/`PSK_FINISH`.

Refer to the [Requester Guide](/guides/requester) for the full flow, or see [`spdm-emu`](https://github.com/DMTF/spdm-emu) for a complete working example.
